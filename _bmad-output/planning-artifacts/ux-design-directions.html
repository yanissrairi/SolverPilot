Direction 1: Dense Power User

High Density List-Based Information First
Philosophy: Maximum information density for power users. Compact spacing (py-1, 4px vertical), alternating row backgrounds, minimal chrome. Optimized for 15-18 visible jobs without scrolling. Shows chunking dividers every 10 jobs for 20+ queues.

Best for: Users who want to see as much status at once as possible. Minimal distraction, maximum data.

Direction 2: Balanced Clarity

Balanced Density Glassmorphism Current Alpha+
Philosophy: Evolution of current Alpha design with refinements from visual foundation. Balanced spacing (py-2, 8px vertical), clear visual hierarchy through panel opacity (85%/75%/80%), enhanced status badges. Performance-optimized blur (2px panels, 12px header).

Best for: Natural progression from Alpha. Familiar to existing users, adds queue features without radical redesign.

Direction 3: Status-First

Balanced Density Visual Emphasis Scannable
Philosophy: Status badges are the primary visual element. Larger badges (14px font, 6px padding), stacked layout with status on top. Quick visual parsing through prominent color-coded indicators. Vertical layout shows more metadata per job.

Best for: Users who primarily monitor status changes and need to quickly spot running/failed jobs at a glance.

Direction 4: Timeline View

Timeline Gantt-Style Progress-First
Philosophy: Queue visualized as a timeline/Gantt chart. Horizontal progress bars show elapsed time and completion progress. Left border color indicates status. Full-width single-panel view prioritizes temporal relationships.

Best for: Users who think in terms of time and want to see relative duration and progress of jobs.

Direction 5: Card-Based

Cards Spacious Modern UI
Philosophy: Each job as an independent card with hover interactions. Grid layout adapts to screen width. More white space, cleaner aesthetic. Each card is a self-contained unit with all metadata visible.

Best for: Users who prefer modern UI patterns and want to interact with jobs as discrete objects. Good for smaller queues (<20 jobs).

Direction 6: Minimal Command Line

Terminal Ultra-Minimal CLI-Style
Philosophy: Terminal-inspired interface with text-based visualization. Monospace only, no glassmorphism, black background. Commands and output mimic CLI tools. Maximum authenticity for command-line power users.

Best for: Hardcore terminal users who want GUI convenience but CLI aesthetics. Most authentic "tmux with a GUI" interpretation.
